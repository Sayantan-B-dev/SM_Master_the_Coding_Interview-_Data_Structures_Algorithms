# ZTM Discord Community — Important Information

## Core Purpose

The ZTM Discord server acts as the **primary operational hub** for:

- Communication  
- Doubt resolution  
- Networking  
- Course updates  
- Career-related discussions  

It is tightly integrated with the course workflow and should be treated as an **active learning layer**, not a passive resource.

---

| Channel Name              | Functional Role                                                                 |

## Communication Protocol

### Message Structure Best Practices

- Clearly define your problem statement  
- Include:
  - Code snippet (if applicable)  
  - Expected output  
  - Actual output  
  - Attempted approach  

#### Example

```text
Problem: Binary search not returning correct index  
Input: [1,2,3,4,5], target = 4  
Expected: 3  
Actual: -1  
Approach: Iterative binary search implementation  
````

This improves response quality and reduces resolution time.

## Accountability System

### Purpose

* Enforces consistency through external monitoring
* Reduces drop-off rate during long courses

### Implementation

* Pair with one or more learners
* Define:

  * Daily or weekly goals
  * Check-in frequency
  * Shared milestones

### Example Structure

```text
Daily Goal: Solve 3 DSA problems  
Weekly Goal: Complete 1 module  
Check-in: Every night at 10 PM  
```

## Critical Rules for Effective Usage

* Do not ask vague or incomplete questions
* Avoid spamming multiple channels for the same query
* Search previous messages before asking
* Maintain professional and respectful communication
* Contribute back to the community when possible
